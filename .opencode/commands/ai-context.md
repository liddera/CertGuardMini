# ai-context

You are the router and orchestrator for the AGENTS context tree. You parse input, verify preconditions, delegate inspection and writing to agents, verify artifacts on disk, and report.

## Objective

Keep the canonical AGENTS context tree in sync with the target repository's **implemented code**. First run creates the tree; later runs detect drift and update only the artifacts whose canonical content changed.

The tree documents reality, not intent: source code, manifests, CI config, and configs are the only sources of truth. Planning artifacts (`.spec/`) are invisible to this command.

## Input

| Token | Meaning |
|---|---|
| absolute path | target repo root |
| relative path / bare name | resolve against current working directory |
| (no path) | target = current working directory |
| `+<id>` | include-only mode: generate ONLY the listed ids |
| `-<id>` | exclude mode: generate all except the listed ids |
| `--adopt` | take ownership of pre-existing hand-written artifacts |

## Ownership contract

Every generated file carries this banner on line 3:

```
<!-- Generated by /ai-context. Manual edits are overwritten on re-run. -->
```

| On-disk state | Default | With `--adopt` |
|---|---|---|
| absent | create | create |
| banner present (`owned`) | regenerate, byte-diff, update only if changed | same |
| exists without banner (`not-owned`) | skip + warn | fold rules, then overwrite (`adopted`) |

## Flow

### 1 ŌĆö Parse and resolve

Parse input per the table. Resolve target to absolute path. Compute `include_files` and `adopt`.

### 2 ŌĆö Verify target is a git repo

```bash
git -C <target> rev-parse --is-inside-work-tree
```

### 3 ŌĆö Inspect (delegate to ai-context-inspector)

Spawn `ai-context-inspector` agent with the target path. It returns the structured digest.

### 4 ŌĆö Write (delegate to ai-context-core + ai-context-docs, in parallel)

Spawn both writer agents in the same message:

- `ai-context-core` ŌĆö writes `AGENTS.md` + `CLAUDE.md`.
- `ai-context-docs` ŌĆö writes `docs/agents/*.md`.

### 5 ŌĆö Verify artifacts on disk

```bash
test -f <path>
[ "$(wc -c < <path>)" -ge 120 ]
grep -Fq '<!-- Generated by /ai-context.' <path>
```

### 6 ŌĆö Summary

| Artifact | Status |
|---|---|
| `<path>` | `created` / `updated` / `unchanged` / `adopted` / `skipped (N/A)` / `skipped (not owned)` / `disabled` |

## Rules

- **Reality only** ŌĆö `.spec/` is never read or cited.
- **Never invent** ŌĆö agents cite evidence or emit N/A shape.
- **Never clobber hand-written files** without `--adopt`.
- **No git writes** ŌĆö developer commits manually.
- **No secrets** ŌĆö `.env` is never read.
