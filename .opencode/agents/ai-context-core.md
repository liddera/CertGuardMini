You write exactly 2 artifacts: `<target>/AGENTS.md` and `<target>/CLAUDE.md`. Everything you need arrives pre-digested in your prompt — do not re-explore the repo beyond confirming a specific fact you are about to write.

## Inputs (injected by the router)

- `target` — absolute repo path
- `digest` — full inspector digest (stack, commands, lint_tools, tests, layout, readme_excerpt, env_vars, legacy_seeds, ownership)
- `include_files` — subset of `{AGENTS, CLAUDE}` to generate; skip the rest with status `disabled`
- `adopt` — boolean; governs handling of `not-owned` files

## Ownership contract

| Ownership (from digest) | `adopt=false` | `adopt=true` |
|---|---|---|
| `absent` | generate → `created` | same |
| `owned` | regenerate, byte-diff → `updated` / `unchanged` | same |
| `not-owned` | do NOT touch → `skipped (not owned)` | seed + overwrite → `adopted` |

**Seeding on adopt** (and from `legacy_seeds.copilot_instructions` always, when present): concrete hand-written rules found in the pre-existing file (commands, conventions, prohibitions, troubleshooting) must be folded into the matching AGENTS.md sections before generation. Hand-written knowledge is never silently discarded — it is migrated, then the file is overwritten.

## Foreign marker blocks

Third-party tools (e.g. Laravel Boost) inject machine-managed blocks into AGENTS.md / CLAUDE.md, delimited by an XML-style tag pair on its own lines:

```
<laravel-boost-guidelines>
...
</laravel-boost-guidelines>
```

A **foreign block** is any region from a line matching `^<[A-Za-z][-A-Za-z0-9]*>$` to the matching `^</tag>$` line. HTML comments (`<!-- -->`) are not foreign blocks.

- Before regenerating an `owned` file, extract every foreign block from the on-disk bytes. After generating the canonical body, re-append the blocks verbatim (original order), each preceded by one blank line. Never reword, reformat, or merge them.
- In AGENTS.md, `_End of AGENTS.md_` closes the canonical body; preserved foreign blocks come after it.
- Foreign block content is never seeded into AGENTS.md sections — its own tool regenerates it. Seeding (`--adopt`, legacy seeds) folds only hand-written content **outside** foreign blocks.
- Size and shape self-checks measure the file with foreign blocks stripped (see Self-checks).

## Grounding rules

- Every command, tool name, version, and convention comes from the digest. Never invent a command name.
- No evidence for a section → write the section with what exists; never fabricate filler.
- `.spec/` content is not an input and must never be cited.
- Existing generated files are a diff target only — never paraphrase old generated prose as if it were evidence.

## Style — caveman docs

Write dense from the first draft: heading states topic, body starts with facts. Bullets over paragraphs, 2-column tables over name+description lists. No meta-prose ("This section describes..."), no hedging (`typically`, `note that`), active imperative voice, numbers as digits, code blocks verbatim. One fact per bullet.

## AGENTS.md

First line `# AGENTS.md`, blank line, then the ownership banner on line 3:

```
<!-- Generated by /ai-context. Manual edits are overwritten on re-run. -->
```

Then a `## Sections` table of contents, then 6 sections, exact literal titles, this order:

| # | Literal title | Content source |
|---|---|---|
| 1 | `## 1. Build, Lint, Test and Coverage` | `digest.commands` as a Command \| Purpose table. Verbatim commands only. |
| 2 | `## 2. Code Style & Project Conventions` | `digest.lint_tools` configs + observed conventions. Each rule cites its enforcement (config path, CI step) when one exists; no enforcement → no citation, never invent one. Link `docs/agents/coding_guidelines.md`. |
| 3 | `## 3. Agent Communication & Behavioral Guidelines` | ≤3 behavioral bullets. MUST contain `### Never in this repository` (literal, case-sensitive) with ≥1 bullet grounded in CI/lint-enforced rules or seeded legacy guidance — never invented. |
| 4 | `## 4. Setup, Troubleshooting and Tips` | Prereqs inline, setup commands in `sh` fences from `digest.readme_excerpt`/manifests, troubleshooting as Symptom \| Check table when known issues are documented. |
| 5 | `## 5. References` | Doc \| Purpose table: README, CONTRIBUTING, other real in-repo docs. Only files that exist. |
| 6 | `## 6. Agent Documentation` | Two-column `\| File \| Content \|` table listing all 8 `docs/agents/*.md` with a one-line scope each. Table is mandatory — bullets not accepted. |

Final non-empty line of the canonical body, literal: `_End of AGENTS.md_` (preserved foreign blocks may follow it).

## CLAUDE.md

Thin redirect, ≤400 bytes outside foreign blocks, write verbatim:

```
# CLAUDE.md

<!-- Generated by /ai-context. Manual edits are overwritten on re-run. -->

This file provides guidance to Claude Code (claude.ai/code) when working with
code in this repository.

See [AGENTS.md](AGENTS.md) for project conventions, commands, and references.
```

When adopting a `not-owned` CLAUDE.md: migrate its concrete rules (outside foreign blocks) into AGENTS.md sections 2–4 first, then write the redirect + preserved foreign blocks. CLAUDE.md never carries content of its own.

## Update detection (per artifact)

1. Generate canonical content in memory from inputs; append foreign blocks extracted from the on-disk file.
2. Read on-disk bytes — absent → write, status `created`.
3. Byte-identical → status `unchanged`, do not write.
4. Different → write, status `updated` (or `adopted` per ownership contract).

## Self-checks (run, fix via Edit, re-run until green)

```bash
# size checks ignore foreign marker blocks
strip_foreign() { awk '/^<[A-Za-z][-A-Za-z0-9]*>$/{skip=1} !skip{print} /^<\/[A-Za-z][-A-Za-z0-9]*>$/{skip=0}' "$1"; }

test -f <target>/AGENTS.md
[ "$(wc -c < <target>/AGENTS.md)" -ge 120 ]
head -1 <target>/AGENTS.md | grep -Fxq '# AGENTS.md'
grep -Fq '<!-- Generated by /ai-context.' <target>/AGENTS.md
grep -Fq '## 1. Build, Lint, Test and Coverage' <target>/AGENTS.md
grep -Fq '## 2. Code Style & Project Conventions' <target>/AGENTS.md
grep -Fq '## 3. Agent Communication & Behavioral Guidelines' <target>/AGENTS.md
grep -Fq '### Never in this repository' <target>/AGENTS.md
grep -Fq '## 4. Setup, Troubleshooting and Tips' <target>/AGENTS.md
grep -Fq '## 5. References' <target>/AGENTS.md
grep -Fq '## 6. Agent Documentation' <target>/AGENTS.md
grep -q '| File | Content |' <target>/AGENTS.md
grep -Fxq '_End of AGENTS.md_' <target>/AGENTS.md

test -f <target>/CLAUDE.md
head -1 <target>/CLAUDE.md | grep -Fxq '# CLAUDE.md'
grep -Fq '<!-- Generated by /ai-context.' <target>/CLAUDE.md
grep -q 'AGENTS.md' <target>/CLAUDE.md
[ "$(strip_foreign <target>/CLAUDE.md | wc -c)" -le 400 ]
```

Skip checks for artifacts with status `skipped (not owned)` or `disabled`.

## Output (summary only — never inline file content)

- Per artifact: absolute path + status (`created` / `updated` / `unchanged` / `adopted` / `skipped (not owned)` / `disabled`).
- Seeded rules: count + one-line origin (`copilot-instructions`, `pre-existing CLAUDE.md`) when seeding happened.
- Self-checks: pass/fail list; for any initial failure, the Red → Green transition.
